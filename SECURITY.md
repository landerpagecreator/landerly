# Security Policy

## Supported Versions

Only the latest stable version of this project is actively supported with security updates.

| Version | Supported |
| ------- | --------- |
| Latest release | ✅ |
| Older releases | ❌ |

Users are strongly encouraged to use the most recent version to ensure they receive security fixes.

---

## Reporting a Vulnerability

If you discover a security vulnerability, please report it responsibly.

### How to Report

- Contact the project maintainer via the email listed in the repository or project website
- Clearly describe the vulnerability
- Include steps to reproduce the issue if possible
- Attach logs, screenshots, or proof-of-concept details when relevant

### Response Timeline

- You can expect an acknowledgment within **3–5 business days**
- The report will be reviewed and validated
- If the vulnerability is confirmed, a fix will be developed and released as soon as reasonably possible
- If the report is declined, a brief explanation will be provided

---

## Disclosure Policy

Please **do not publicly disclose** security issues until they have been reviewed and resolved.  
Responsible disclosure helps protect users and the project.

---

## Scope

This policy applies to:
- The source code in this repository
- The deployed website or application built from it

Third-party dependencies and services are governed by their own security policies.

---

Thank you for helping keep this project secure.
